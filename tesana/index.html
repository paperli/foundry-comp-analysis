Competitive Analysis—Tesana

Snapshot — what Tesana is

Tesana is a generative-AI “make any game” platform powered by its own proprietary model, Muranyi-3, which turns a text prompt into a complete, playable game — assets, mechanics, code, UI and audio — emitted as a real Godot project in minutes, no code or art skills. It pairs creation with a community storefront / feed (Explore) for discovering, playing, favouriting and remixing games, plus a creator back-office (My Games, Profile, Earnings) and a live creator-payout program. Critically vs Astrocade it generates real, exportable engine source (Godot; macOS / Windows / Web / source export) and targets 3D — yet like Astrocade it is screen-first and solo-async-social: no TV mode, no couch / local multiplayer, and voice is a prompting input, not a way to play together.

01

Company, Market & Funding

Who is behind Tesana, what is actually known about funding and scale, and who they are really for.

Founders & team

Co-founders Johannes Vermandois (CEO; Y Combinator alum, prior skincare & AI-marketing startups) and Eric Heimer (CTO). Vermandois demoed Tesana live at GDC in March 2026. Team size undisclosed; the iOS app ships under Dreamsands Inc. A small, founder-led startup — not a game studio or a frontier-AI lab.

Funding & investors

Not publicly disclosed as of June 2026. Press (PC Gamer) explicitly notes the team, tech and funding details are “thin.” No round, lead, angel or valuation on record — a stark contrast to Astrocade’s loud $56M / Sequoia + Google + Nvidia story.

Traction

One hard public number: ~10,000 paying customers within the first ~2 weeks of launch (late Mar 2026). MAU, games created, plays and creator counts are all undisclosed. Growth claims are founder-stated, not third-party verified.

Positioning & market

Tagline “The #1 AI Game Maker.” Mission: bring 100M new people into game-making — explicitly aimed at non-coders. Frames AI-made indie games as the future. Named rivals: Roblox, Rosebud, Ludo.ai; open-source comparator Godogen.

02

Gameplay Platform

The games themselves and how people actually play them.

Game types

Real-time 2D and 3D games across many genres: platformers, FPS / shooters, racing, survival sandbox, roguelike, RPG, simulation, strategy, cozy and story games. Deeper and more varied than Astrocade’s bite-size 2D mini-games — Muranyi-3 targets “3D worlds with game logic built in.”

How it’s played

Plays instantly in the browser (play.tesana.ai) and in iOS / Android apps; exportable to macOS / Windows / Web / Godot source on paid tiers. Mouse / touch / keyboard, single-player. No console / TV target, no gamepad, no local couch co-op.

Discovery feed

“Explore” is a community storefront: “Featured Today” + genre rails (Adventure, etc.). Each tile shows title, creator handle, play count (top games 30K+) and favourite / ▶ play. App-Store-style categories blended with a social feed.

Session & retention loop

Loop: play → favourite / follow → remix → create → publish to feed → share clips to earn. A creation-and-distribution flywheel — but consumed solo on a personal screen, not as a shared in-the-room session.

03

Game-Creation Toolkit & Tools UX

The creator-facing tooling: from blank prompt to a published, exportable game — captured live in the signed-in Studio.

Entry: prompt or template

The Studio opens on “What game should we make now, paperli?” — a multiline prompt box, a row of genre-tagged starter templates (Slice Storm / COZY, Patient Zero / FPS, Jungle Dash / MOBILE…), Style- and Character-reference image uploads, and a model picker (“Smart”). Left rail: Dashboard, My games, Profile, Earn, Explore.

Free vs paid gating

Key finding: on Free, building from a blank prompt is disabled — “Upgrade to build your own game from scratch.” Free users are funnelled into templates / remix; custom-asset import, faster models and all export are paywalled. Credits gate everything.

Iterate by chat + Plan mode

Creation is a chat: describe changes in plain language and the agent regenerates art, code, mechanics and audio in the live game. A “Plan” toggle lets the agent plan before building. Remixing any community game is a first-class on-ramp.

Under the hood: real Godot

The Studio exposes Game / Assets / Animations / VFX / Code tabs. The Code tab reveals an actual Godot project — project.godot, .tscn scenes, GDScript (.gd), an assets/ tree, even a CLAUDE.md: genuine, exportable engine source.

★ Key feature — real engine output

A genuine Godot project you can own & export

  • Unlike sandbox-only generators, Tesana hands you a real Godot project — .tscn scenes, GDScript, assets.
  • On paid tiers it exports to macOS, Windows, playable Web HTML or raw source — genuine ownership and portability, not platform lock-in.
  • Arguably Tesana’s single biggest wedge against Astrocade, whose games can’t leave the platform.
  • For Weekend / Foundry → match the “you own it, take it anywhere” promise, framed for TV: export a couch-ready build, not just a feed clip.

★ Strategic read — the free-tier funnel

Free gets you templates; creation is the paywall

  • The Free plan deliberately disables blank-prompt creation (“Upgrade to build your own game from scratch”) and funnels new users into one-tap templates and remixing — which DO generate a full game (we built one end-to-end on a $0 account).
  • Scratch builds, custom-asset import, faster models and all export sit behind Pro+. Time-to-first-game is near-zero; open-ended creation and ownership are the conversion levers.
  • For Weekend / Foundry → let players start instantly from templates on the TV, reserving deeper authoring & export for upgrade.

04

Agentic AI Creation UX

Tesana’s signature: a single agent that plans, then generates art, audio, code and logic — the “prompt → playable game” pipeline, with the work shown live.

  1. 1

    Prompt or template

    Describe a game, tap a genre template, or remix a community game. Optionally add style / character reference images.

  2. 2

    Plan (optional)

    A “Plan” toggle lets the agent outline the build first; otherwise it goes straight to generating.

  3. 3

    Agent builds live

    One agent streams the work: sprites, backgrounds, SFX and music, then the game code (GDScript).

  4. 4

    ~Minutes — watched, not hidden

    A live thought-stream (“Generating Watermelon…”, “Programming swipe detection”) — not a ~10-min black box.

  5. 5

    Play, iterate, publish

    Play in-pane, refine by chat, then publish to the feed or export; remixable by others.

Art & assets

Generates sprites, textures, backgrounds and 3D models from your prompt (21 assets for our fruit game), guided by optional reference images.

Game logic & code

Writes real GDScript — input / swipe detection, physics, spawning, scoring, win / lose — as a working Godot project, not a snippet.

Audio

Auto-composes music and sound effects (Slice, Splat, Bomb Explosion, Combo Chime) tuned to the game.

Orchestration · Muranyi-3

The proprietary model plans & coordinates the build — reportedly orchestrating external LLMs (incl. Claude) + asset generators behind the scenes.

05

Social Factors

Community, identity and the loops that make creation social.

Creator profiles

Every game shows a creator handle (@juicer, @sharky…) linking to a profile / portfolio. Identity is creator-first — closer to YouTube / TikTok creators than gamer gamertags. Follow + Following are built in.

Remix culture

Any game can be remixed as a starting point (“Remix game” on every tile); Max+ can even “disable remixing.” Remix is the default on-ramp — compounding content and giving newcomers a fast start.

Community is off-platform

The “Community” surface is a Discord invite, plus Help and Getting-started. Like Astrocade, the real social graph — feedback, tips, support — lives on Discord, not in-app.

Favourites & ranked feed

Favourites, Following and a play-count-ranked Explore feed drive discovery; social proof (30K+ plays on top games) is front-and-centre. But interaction is async and solo, never co-present.

06

Business Model & Creator Economy

How the platform and its creators make money.

How the platform earns

Credit-based subscriptions: Free, Pro $20, Max $50, Ultra $200 / mo (2.5K → 35K credits) plus one-time top-ups; credits refresh on a timer on Free. Each generation burns GPU / compute — high marginal cost is the core risk (as with Astrocade).

How creators earn

A live payout program: creators are paid for verified game plays AND for social clips of their game (TikTok / Reddit / IG / YouTube / Facebook) per 1K views. Cashed out weekly via Stripe above $25.

The viral loop

Paying creators to post clips off-platform turns every creator into a paid acquisition channel — a UGC growth flywheel. Astrocade pays on in-app plays; Tesana extends payouts to off-platform clip views.

Ownership as upsell

Commercial rights, watermark removal, source-code download and macOS / Win / Web export are Pro+ perks. “You own your game” is both the value-prop and the primary upgrade lever.

Strengths & Weaknesses

What Tesana does genuinely well, and where it is exposed.

Strengths

  • Real, exportable Godot output — genuine ownership, not lock-in (its biggest edge vs Astrocade)
  • 2D and 3D across many genres — more depth than bite-size mini-games
  • Near-zero time-to-first-game via templates / remix
  • Transparent live build thread — you watch assets & code generate
  • Multi-tab Studio: Game · Assets · Animations · VFX · Code
  • Clip-to-earn payouts turn creators into a viral acquisition channel
  • Clean, fast, modern UX; instant browser + mobile play

Weaknesses

  • Free tier blocks scratch creation — can dampen first-run activation
  • Information-opaque: no funding, MAU or scale figures disclosed
  • Very early (launched ~Mar 2026); durability unproven
  • AI output quality varies with the prompt; templates are safest
  • Heavy GPU cost; credits feel restrictive (mixed reviews on price)
  • Community outsourced to Discord; thin in-app social graph
  • Screen-only: no TV, no couch co-op, no voice-first play

User Stories & Jobs-to-be-Done

Read backwards from the features we actually used — the stories Tesana’s tooling is built to fulfil, by persona.

★ North-star job-to-be-done

Turn an idea into a real, playable — and ownable — game in one sitting

“As someone with a game idea but no coding or art skills, I want to turn it into a real, playable, shareable game I can keep and even sell — in one sitting.” The deeper job: collapse the consumer → creator gap so the audience becomes the content engine — and, uniquely vs Astrocade, hand them something they truly own.

1 · Aspiring creator (non-coder)

  • As a non-coder, I want to describe (or just pick) a game and get something playable, so creating needs a sentence, not code.
  • As a first-timer, I want a template to start from, so I skip the blank page.
  • As someone impatient, I want to watch it build, so the wait feels alive.

2 · Tinkerer / iterator

  • As someone refining fun, I want to change the game by chatting, so I improve it without restarting.
  • As a careful builder, I want Plan mode, so I review before it builds.
  • As a remixer, I want to fork any game, so I learn by modifying.

3 · Owner / shipper

  • As a creator, I want to export to macOS / Windows / Web / source, so I truly own and ship my game.
  • As a seller, I want commercial rights and no watermark, so I can monetise it — free of platform lock-in.

4 · Aspiring earner

  • As a creator, I want to be paid for plays AND for clips I post on social, so reach converts to income.
  • As a data-minded creator, I want play counts on every game, so I double down on what works.
  • Cash out weekly via Stripe.

5 · Player → creator

  • As a player, I want an endless Explore feed of instant games, so it’s a fun way to spend time.
  • As a player who likes a game, I want to Remix it, so consuming naturally tips me into creating.

◆ Strategic read

What the tool is really optimising for

Tesana’s real job isn’t “make a game” — it’s to collapse the consumer → creator gap AND give creators something they own. It optimises for time-to-first-playable, the confidence to keep iterating, a reason to come back and earn, and — uniquely vs Astrocade — exportable ownership. A creator-economy playbook (YouTube / TikTok) fused with a “real tool you keep” promise. The craft shifts from building to prompting, tuning and shipping.

★ The anti-stories — what Tesana deliberately doesn’t serve

The unmet stories = the Weekend / Foundry opening

  • “…so my friends and I can play together in the same room, on the TV.” → couch / local multiplayer (game night)
  • “…so I can create and play hands-free, just by talking.” → voice-first interaction (Tesana uses voice only to prompt)
  • “…so we can play licensed shows like Jeopardy or Wheel of Fortune together.” → owned TV-show IP + party formats

Co-present TV play, voice-first interaction and big-IP party games are precisely what Weekend / Foundry is built to own.

09

Implications & Opportunities for Weekend / Foundry

So what? Where Tesana’s shape leaves room for a TV-first, voice-first, local-multiplayer platform.

Our whitespace

Like Astrocade, Tesana is screen-first and solo-async-social. TV gaming, couch / local multiplayer (game night) and voice-first PLAY are exactly what it does NOT do — voice is only a prompt input. That category is ours to own.

Differentiators to own

The TV as the shared screen with phones as controllers. Designed-for-together, in-the-room multiplayer. Voice-first creation AND play. “Game night” occasions and licensed TV-show IP (Jeopardy, Wheel of Fortune…) vs anonymous indie mini-games.

Plays worth borrowing

Template / remix on-ramp + near-zero time-to-first-game. The live agent thought-stream (builds trust). REAL ownership & export. The clip-to-earn viral loop. A multi-tab live-preview editor.

Risks to avoid

Don’t gate first creation so hard it kills activation. Budget per-generation compute early. Don’t ship shallow one-offs — couch play needs depth & replayability. Avoid info-opacity; don’t outsource community to Discord.
